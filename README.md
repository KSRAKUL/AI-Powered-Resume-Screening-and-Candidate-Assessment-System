## 🏷️ 5-Tier Fit Classification System

The AI evaluates each candidate and assigns exactly one of five categories:

| Tier | Classification | Criteria | Badge |
|:----:|----------------|----------|:-----:|
| 1 | **Not Aligned** | Only basic/generic skills; no meaningful overlap with the role | 🔴 |
| 2 | **Needs Heavy Training** | Lacks more than 40% of core skills but shows potential for growth | 🟠 |
| 3 | **Core Skills Match (Trainable)** | Has 50–70% of core foundational skills; training can bridge remaining gaps | 🟡 |
| 4 | **Strong Fit (Plug-and-Play)** | Possesses 70–90% of required skills; ready to contribute immediately | 🔵 |
| 5 | **Outstanding Expert** | Exceeds role requirements with deep, expert-level alignment across all areas | 🟢 |

Each tier maps to a distinct color gradient in the UI's radial score gauge for instant visual identification.

The application uses a **fully decoupled client–server architecture**:

- **Frontend** sends a `FormData` POST (PDF file + job description text) to the backend
- **Backend** extracts text, matches skills, calls the local LLM, and returns structured JSON
- **LLM** runs entirely on your machine via Ollama — no data ever leaves your network

## 🚀 Getting Started

### Prerequisites

| Tool | Minimum Version | Download |
|------|:-:|------|
| Python | 3.10+ | [python.org](https://www.python.org/downloads/) |
| Node.js | 18+ | [nodejs.org](https://nodejs.org/) |
| Ollama | Latest | [ollama.com](https://ollama.com/download) |

### Step 1 — Clone the Repository

```bash
git clone https://github.com/DhanushS-11/AI-Powered-Resume-Screening-and-Candidate-Assessment-System.git
cd AI-Powered-Resume-Screening-and-Candidate-Assessment-System
```

### Step 2 — Install & Start the Local LLM

```bash
# Pull the required model (one-time download, ~1.9 GB)
ollama pull qwen2.5:3b
```

> **Verify:** Visit `http://localhost:11434` in your browser — it should display *"Ollama is running"*.

### Step 3 — Set Up the Backend

```bash
# Create and activate a virtual environment
python -m venv venv

# Windows
venv\Scripts\activate
# macOS / Linux
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Start the FastAPI server (auto-reloads on file changes)
cd backend
python main.py
```

> The API will be available at **http://localhost:8000**

### Step 4 — Set Up the Frontend

Open a **new terminal** window:

```bash
cd frontend

# Install Node.js dependencies
npm install

# Start the Next.js dev server
npm run dev
```

> The UI will be available at **http://localhost:3000**

### Step 5 — Start Screening 🎉

Open **[http://localhost:3000](http://localhost:3000)** in your browser, upload a PDF resume, paste a job description, and click **Analyze Resume Qualifications**.

## 📡 API Reference

### `GET /api/health`

Check server status and Ollama connectivity.

```json
// Response
{
  "status": "ok",
  "ollama_online": true
}
```

### `POST /api/analyze`

Analyze a resume against a job description.

**Request** — `multipart/form-data`

| Field | Type | Required | Description |
|-------|------|:--------:|-------------|
| `resume` | File (PDF) | ✅ | Candidate's resume |
| `job_description` | string | ✅ | Full job description text |

**Response** — `application/json`

```json
{
  "success": true,
  "filename": "jane_doe_resume.pdf",
  "score": 8.0,
  "category": "Strong Fit (Plug-and-Play)",
  "resume_skills": ["Python", "React", "Docker", "AWS"],
  "missing_skills": ["Kubernetes", "Terraform"],
  "ai_analysis": "## Candidate Summary\nJane is a ...",
  "resume_text": "Full extracted text from PDF..."
}
```

| HTTP Code | Meaning |
|:---------:|---------|
| `200` | Analysis completed successfully |
| `400` | Invalid file type or empty/scanned PDF |
| `500` | Internal processing error |

## 🔄 How It Works

Each resume goes through a **4-stage processing pipeline**:

### 1️⃣ Text Extraction

The uploaded PDF is parsed page-by-page using `pdfplumber`. All extracted text is concatenated and forwarded to both the skill matcher and the LLM for contextual analysis.

### 2️⃣ Text Preprocessing

Raw text is cleaned through:
- Conversion to lowercase
- Removal of special characters via regex (`[^a-zA-Z0-9 ]`)
- Whitespace consolidation

This creates a normalized representation for accurate keyword matching.

### 3️⃣ Skill Extraction & Gap Analysis

The preprocessed resume and job description are scanned against a curated database of **27 technical skills** (in `backend/data/skills.txt`). The engine identifies:

| Output | Description |
|--------|-------------|
| **Matching Skills** | Skills found in *both* the resume and the job description |
| **Missing Skills** | Skills required by the JD but *absent* from the resume |

### 4️⃣ AI Assessment Generation

All data — resume text, job description, matched skills, and gaps — is formatted into a structured prompt and sent to the local Ollama instance. The LLM returns a comprehensive assessment with:

| Section | What It Contains |
|---------|-----------------|
| 🎯 **Candidate Score** | Numerical rating `/10` with one-sentence justification |
| 🏷️ **Fit Classification** | One of 5 hiring tiers (Not Aligned → Outstanding Expert) |
| 📝 **Candidate Summary** | Brief professional overview of the candidate |
| 📊 **Match Assessment** | Detailed analysis of role fit |
| ✅ **Key Strengths** | Top competencies and experience highlights |
| ⚠️ **Skill Gaps** | Specific areas requiring development or training |
| 💡 **Hiring Recommendation** | Actionable recommendation for hiring managers |
| ❓ **Interview Questions** | 5 targeted questions to probe identified skill gaps |

> The backend extracts the **score** and **classification** from the LLM's markdown using regex, then sends the full narrative to the frontend's `MarkdownRenderer` for visual display in a card-based grid layout.

## 🎨 Design Language

The frontend follows a **"Space-Dark"** design system:

| Element | Implementation |
|---------|---------------|
| **Background** | Deep slate (`#0B0F19`) with ambient indigo/violet/emerald gradient orbs |

| **Interactions** | Hover lifts, glow borders, drag-and-drop visual feedback, spinner states |
| **Layout** | Responsive — single column on mobile, multi-column grid on desktop |
